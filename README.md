# 口播智能粗剪工具

一个面向口播短视频的 Web 剪辑工具。

目标不是做复杂时间线剪辑，而是把常见口播流程做成一条清晰链路：

- 上传视频
- 生成字幕
- 根据字幕逐字删片
- 左侧实时预览粗剪结果
- 导出可发布的 `MP4`

当前技术栈：

- `Next.js 16`
- `React 19`
- `Tailwind CSS`
- `ffmpeg`

---

## 项目定位

这个项目主要解决口播视频里的几类重复劳动：

- 删除停顿、气口、语气词
- 按字幕而不是按时间轴删内容
- 统一倍速和音量
- 导出带字幕的竖屏成片

适用场景：

- 知识口播
- 短视频讲解
- 个人 IP 表达类视频
- 需要快速粗剪再微调的内容工作流

---

## 当前功能

### 1. 上传视频

- 支持本地视频文件上传
- 自动读取文件名、大小、时长

### 2. 生成字幕

- 当前支持豆包语音识别优先
- 如果云端链路不可用，保留本地回退逻辑，保证项目可运行

### 3. 字幕驱动剪辑

- 右侧按字幕逐字删除
- 支持拖动选中字后批量删除
- 支持恢复单字、恢复整行、重置
- 支持 AI 建议删除片段

### 4. 粗剪预览

- 左侧小尺寸竖屏预览
- 自动跳过已删除片段
- 支持字幕字号、倍速、音量设置

### 5. 导出视频

- `预览导出`：`720 × 960`，更快，用于检查结果
- `最终导出`：`1080 × 1440`，用于发布
- 支持真实导出进度条
- 保留硬字幕导出能力
- 硬件编码优先，失败时回退到 CPU 编码

---

## 当前导出策略

导出分成两档：

### 预览导出

- 目标：快速查看结果
- 输出：`MP4`
- 分辨率：`720 × 960`
- 特点：速度优先，不做整条视频正式硬字幕烧录

### 最终导出

- 目标：平台发布
- 输出：`MP4`
- 分辨率：`1080 × 1440`
- 特点：保留硬字幕压入

当前编码策略：

- macOS 优先：`h264_videotoolbox`
- NVIDIA 优先：`h264_nvenc`
- 回退：`libx264`

## 页面结构

页面按照实际使用顺序拆成 3 个主要区域：

### 上传区

- 负责选择视频和展示基础摘要

### 剪辑区

- 左侧：粗剪预览
- 右侧：字幕逐字编辑

### 导出区

- 负责导出档位选择
- 展示真实进度
- 下载最终文件
